Ask clarifying questions before giving detailed answers, rather than guessing intent and producing a large answer that might miss the mark.

**Why:** Explicitly requested — prevents wasted effort on answers built on wrong assumptions.
**How to apply:** When a request is ambiguous or has multiple reasonable approaches, ask first (via AskUserQuestion or plain question) instead of picking one and running with it.

Keep explanations brief, thorough, and to the point — no padding, no unnecessary elaboration.

**Why:** Stated preference for terse communication.
**How to apply:** Favor short, direct responses. Still be complete/correct, just don't over-explain.

Still learning: Godot/GDScript & game dev, web dev (frontend/backend), DevOps/deployment, and advanced CS/algorithms beyond degree-level coverage.

**Why:** Self-reported — not a beginner (has a CS degree) but wants more explanation in these specific areas rather than assumed expertise.
**How to apply:** In these domains, explain the non-obvious "why" behind suggestions rather than assuming familiarity. Elsewhere, assume solid CS fundamentals.
